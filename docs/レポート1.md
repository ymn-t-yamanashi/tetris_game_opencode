# レポート1: 現状の問題点と改善ポイント

## 1. コンパイル・実行時に発生するエラー
- `Tetris.Tetromino.move_left/1`、`move_right/1` が未定義（LiveViewで呼び出し）。
- `Tetris.GameController` モジュールが存在せず、落下・回転・行消し等のロジックが欠如。
- LiveView の `handle_info(:tick, socket)` でイベントを送信しているだけで実際にゲーム状態を更新していない。

## 2. データ構造の不備
- **GameField**: `grid` は初期化されているが、セルは `nil` / `:filled` しか扱わず、衝突判定ロジックが未実装。
- **Tetromino**: 形状の相対座標配列が無く、`random/0` がシェイプのみ返すだけ。位置・回転情報も不足。
- **GameState**: `current` と `queue` はあるが、キュー生成に不完全な `Tetromino.random/0` を利用している。

## 3. LiveView の不整合
- `render/1` が単にフィールドサイズを表示。実際の盤面描画ロジック未実装。
- キーイベントは `phx-keydown` で送信される想定だが、ハンドラ側で正しく処理できていない。

## 4. テスト不足
- 現在テストファイルが存在せず、ユニットテストとLiveViewインタラクションテストが未実装。

## 5. 改善ロードマップ
1. **GameController** を実装し、`valid_position?/3`, `drop/1`, `rotate/2`, `clear_lines/1`, `lock_piece/1`, `game_over?/1` 等を提供。\n2. **Tetromino** に移動・回転関数を追加し、形状の相対座標配列を保持。\n3. LiveView の `handle_info(:tick, socket)` で `GameController.drop/1` を呼び、状態更新して assign。キーイベントハンドラも修正。\n4. フィールドとテトリミノの描画ロジックを HEEx に実装。\n5. ユニットテスト（GameField, Tetromino, GameState, Score）と LiveView テストを追加。

## 6. 次のステップ
- 上記モジュール・関数を順次実装し、`mix test` が通るようにする。\n- LiveView のビジュアル表示を実装してブラウザで動作確認。
