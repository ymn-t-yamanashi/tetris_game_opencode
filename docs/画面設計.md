# テトリス LiveView 画面設計

## 1️⃣ 全体構成
- **Layout**: `MyAppWeb.Layouts.app` をベースにし、ゲーム画面を `<div id="tetris-container" phx-hook="TetrisHook">` 内に配置。
- **GameView** (`TetrisWeb.GameLive`): LiveView でゲームロジックと描画を行う。`assigns` に `field`, `current_piece`, `score`, `level`, `game_over?` を保持。
- **Components**:
  - `<.grid>`: フィールドのセルを描画するコンポーネント。Tailwind CSS のユーティリティクラスで色分け。
  - `<.piece>`: 現在落下中のテトリミノを描画。
  - `<.info-panel>`: スコア・レベル・次ピース表示。

## 2️⃣ レイアウト詳細
```
<Layouts.app flash={@flash} current_scope="tetris">
  <div class="flex flex-col items-center justify-center h-screen bg-gray-900 text-white">
    <!-- タイトルとスコアバー -->
    <header class="mb-4 w-full flex justify-between px-6 py-2 bg-gray-800">
      <h1 class="text-xl font-bold">TETRIS</h1>
      <div class="flex space-x-4">
<span>Score: {@score}</span>
<span>Level: {@level}</span>
      </div>
    </header>

    <!-- ゲームボード -->
    <main class="grid grid-cols-10 gap-px bg-gray-700" style="width: 400px; height: 800px;">
<%= for {row, y} <- Enum.with_index(@field.grid) do %>
  <div class="flex w-full" phx-update="ignore">
    <%= for {cell, x} <- Enum.with_index(row) do %>
      <.grid cell={cell} x={x} y={y} />
    <% end %>
  </div>
<% end %>
    </main>

    <!-- 次ピースとオーバーレイ -->
    <aside class="mt-4 w-full flex justify-center">
      <.piece piece={@next_piece} position={{-2, -1}} />
    </aside>
  </div>
</Layouts.app>
```

## 3️⃣ イベントハンドリング
- キーイベントは `handle_event("key", %{"code" => code}, socket)` で受け取り、`GameController.move_left/1` 等を呼び出す。
- フレームごとの落下は `:timer.send_interval/2` を LiveView の `mount/3` で設定し、`handle_info(:tick, socket)` 内で `GameController.drop(state)` を実行。

## 4️⃣ コンポーネント設計
### `<.grid>`
```elixir
defmodule TetrisWeb.Components.Grid do
  use Phoenix.Component

  @doc """
  Renders a single cell.
  `cell` is either nil or :filled; style accordingly.
  """
  def grid(assigns) do
    ~H[<div class={grid_class(@cell)}></div>]
  end

  defp grid_class(nil), do: "w-10 h-10 bg-gray-900 border border-gray-800"
  defp grid_class(:filled), do: "w-10 h-10 bg-blue-500 border border-gray-800"
end
```
### `<.piece>`
```elixir
defmodule TetrisWeb.Components.Piece do
  use Phoenix.Component

  @doc """
  Renders a tetromino at given position.
  """
  def piece(assigns) do
    ~H[
      <div class="absolute" style={position_style(@x, @y)}>
        <%= for {dx, dy} <- Tetris.Tetromino.shape_coords(@piece), do: %{
          <.grid cell={:filled} x={@x + dx} y={@y + dy} />
        } %>
      </div>
    ]
  end
end
```

## 5️⃣ ユーザーインタラクション
- **左右移動**：`←/→` キーで `move_left/right`。
- **回転**：`↑` キーで `rotate(clockwise)`。SRS のオフセットは JavaScript Hook で検証し、合法なら LiveView に送信。
- **ハードドロップ**：スペースキーで即座に固定。

## 6️⃣ スタイルとレスポンシブ
- Tailwind CSS を利用。デフォルト幅400px×800pxのボードは `sm:` クラスで縮小可能。
- `phx-hook="TetrisHook"` でキーボードフォーカスを自動取得し、フレームごとに `tick` イベントを送信。

## 7️⃣ エラーハンドリング・テストポイント
- LiveView の `handle_event/3` と `handle_info/2` はパターンマッチで失敗時はログに出力し、ゲームオーバーなら `assign(:game_over?, true)`。
- UI では `@game_over?` が真の場合、オーバーレイで「Game Over」と表示し、再スタートボタンを提供。

---
> **Tip**: 画面の描画は最小限に抑え、変更があったセルのみ `phx-update="ignore"` を使って再描画。これによりパフォーマンスが向上します。
