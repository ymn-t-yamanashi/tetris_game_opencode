# テトリスが動かない調査結果

## 1. 現象
- ブラウザで `http://localhost:4000` を開くと、ゲームフィールドは表示されるものの、ブロックが落下・回転しません。
- LiveView が正しく key イベントや tick イベントを受け取っていないように見えます。

## 2. コードレビュー
1. **LiveView** (`game_live.ex`)
   - `handle_info(:tick, socket)` は `push_event(socket, "tick", %{})` を送信しています。
   - `handle_event("key", ...)` で ArrowLeft/ArrowRight の処理は実装済み。
   - テンプレートでは `<div id="game-field">` に phx-hook が設定されていませんでした。
2. **JavaScript**
   - デフォルトの `priv/static/assets/js/app.js` には LiveView の初期化コードがありませんでした。
   - そのため、クライアント側で LiveView と通信する Hook が登録されていませんでした。

## 3. 原因
- **Hook が未設定**: `game_live.ex` のテンプレートに `phx-hook="GameHook"` を付与していないため、サーバーから送られる `tick` イベントを受け取れず、自動落下が機能しません。
- **LiveView 初期化不足**: `app.js` に LiveSocket のインスタンス作成と接続コードが欠如していたため、クライアント側で Hook がロードされませんでした。/

## 4. 修正内容
1. `lib/tetris_game_opencode_web/live/game_live.ex`
   - `<div id="game-field">` に `phx-hook="GameHook"` を追加。
2. `priv/static/assets/js/hooks/game_hook.js`
   - 新規作成: keydown で ArrowLeft/ArrowRight を送信し、サーバーからの `tick` 受信時に ArrowDown イベントを送るロジックを実装。
3. `priv/static/assets/js/app.js`
   - Phoenix と LiveView のインポート追加。
   - Hook を読み込み、LiveSocket を初期化し接続。

## 5. 検証手順
1. `mix deps.get && mix assets.deploy`（または `npm run deploy`）でアセットをビルド。
2. `mix phx.server` でサーバー起動。
3. ブラウザで `http://localhost:4000` を開き、キーボードの左右キーでブロックが移動し、一定時間ごとに自動落下することを確認。

## 6. 今後の注意点
- LiveView の Hook は必ずテンプレート内に `phx-hook` 属性で紐付ける。
- アセットビルド時にエラーが出ないよう、依存ライブラリ(`phoenix`, `phoenix_live_view`)を正しくインポートしているか確認する。
