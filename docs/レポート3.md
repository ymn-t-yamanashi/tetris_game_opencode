# テトリスが動かない調査結果

## 概要
- `TetrisGameOpencodeWeb.GameLive` は `handle_event("key", ...)` でキー入力を受け取り、現在のピースを左右に移動します。
- `mount/3` では `Process.send_after(self(), :tick, 1000)` を設定して毎秒 `:tick` メッセージを自分自身へ送信し、その際に `push_event(socket, "tick", %{})` を呼び出しています。
- しかし、クライアント側で `"tick"` イベントを受け取る JavaScript ハンドラ（`GameHook`）がプロジェクト内に存在していません。
- さらにサーバ側でも `handle_event("tick", ...)` が実装されておらず、ゲームフィールドの更新やピースの落下処理が行われていません。

## 発見した問題点
1. **JavaScript Hook の欠如**
   - `GameLive` で `<div id="game-field" phx-hook="GameHook">` と宣言しているにも関わらず、`
     assets/js/…` やその他のディレクトリに `GameHook` を実装したファイルが存在しません。
   - このためクライアントは「tick」イベントを受け取っても何もしない状態です。

2. **サーバ側での tick 処理未実装**
   - `handle_info(:tick, socket)` は単に `push_event(socket, "tick", %{})` を行うだけで、ゲームロジック（ピースを1セル下へ移動・衝突判定・ライン消去など）を更新していません。
   - その結果、画面上の表示は常に初期状態のままで、キー入力もサーバ側では正しく処理されているものの、クライアントに反映されません。

3. **リソース構成の欠落**
   - 通常 Phoenix プロジェクトには `assets/` ディレクトリが存在し、JavaScript をビルドするための `package.json`, `app.js` などが含まれます。今回のリポジトリではその構成が見当たりません。
   - ビルドツール（esbuild 等）も設定されていないようです。

## 推奨修正案
- **JavaScript Hook の作成**
  ```js
  // assets/js/game_hook.js
  export default {
    mounted() {
      this.el.addEventListener('tick', () => {
        console.log('tick received');
        // TODO: 描画ロジックを実装
      });
    }
  };
  ```
- **ビルド設定の追加**（`mix.exs` に esbuild の依存と `assets/package.json` を配置）
- **サーバ側での tick 処理**
  ```elixir
  def handle_info(:tick, socket) do
    # 例: 現在ピースを1セル下に移動、衝突判定・ライン消去
    new_piece = Tetris.Tetromino.move_down(socket.assigns.current_piece)
    {:noreply, assign(socket, :current_piece, new_piece)}
  end
  ```
- **LiveView のレンダリング** を更新し、`phx-update="stream"` 等を用いて動的にフィールドを描画。

## 現状の制約
- 本リポジトリにはまだビルドパイプラインが構築されていないため、上記 JavaScript をコンパイルしてブラウザへ配信する手順が未整備です。
- まずは `assets/` ディレクトリを作成し、必要な依存（esbuild, phoenix_live_view 等）を追加してください。詳細は Phoenix の公式ドキュメント「Assets」セクションをご参照ください。

---
**結論**：テトリスが動かない主因は、クライアント側の JavaScript ハンドラとサーバ側の `tick` 処理が実装されていないことです。これらを追加し、ビルド環境を整えることでゲームロジックが正常に機能します。
