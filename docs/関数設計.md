# テトリス コアロジック設計

## 1️⃣ 全体構成
- **GameField**: フィールド格子(10×20)の状態管理。空き/埋まりを`boolean`または`nil / :filled`で保持。
- **Tetromino**: テトリミノ型(I,O,T,S,Z,J,L)、回転パターン、現在位置・向きを持つ構造体。
- **GameState**: フィールド + 現在の落下ブロック + 次ブロックキュー + スコア・レベル・ゲームオーバーフラグ。
- **Score**: 行消しごとのスコア計算(`lines * (level+1) * 100`)。
- **GameController**: 入力処理と状態遷移を行う関数群。

## 2️⃣ データ構造（Elixir モジュール）
```elixir
# GameField
defmodule Tetris.GameField do
  @width 10
  @height 20
  defstruct grid: List.duplicate(List.duplicate(nil, @width), @height)
end
```
```elixir
# Tetromino
defmodule Tetris.Tetromino do
  defstruct shape: :i,
            rotation: 0,
            position: {5, 0} # (x,y) of the top-left reference cell
end
```
```elixir
# GameState
defmodule Tetris.GameState do
  defstruct field: %Tetris.GameField{},
            current: nil,
            queue: [],
            score: 0,
            level: 0,
            game_over?: false
end
```
```elixir
# Score (pure function)
defmodule Tetris.Score do
  def calculate(lines, level), do: lines * (level + 1) * 100
end
```
## 3️⃣ 主関数（機能単位）
| 機能 | モジュール | 関数名 | パラメータ | 戻り値 |
|------|-----------|--------|------------|---------|
| フィールド初期化 | GameField | new/0 | – | %GameField{} |
| テトリミノ生成 | Tetromino | random/0 | – | %Tetromino{} |
| 位置合法性チェック | GameController | valid_position?/3 | field, tetro, pos | boolean |
| ブロック落下 | GameController | drop/1 | state | new_state |
| 回転（SRS） | GameController | rotate/2 | state, direction | new_state |
| 行消し判定・削除 | GameController | clear_lines/1 | field | {new_field, lines_cleared} |
| スコア更新 | Score | calculate/2 | lines, level | points |
| レベル計算 | GameState | update_level/1 | state | new_state |
| ゲームオーバー判定 | GameController | game_over?/1 | state | boolean |
| 入力ハンドラ | GameController | handle_event/3 | event, params, socket | {:noreply, socket} |
## 4️⃣ ステート遷移フロー
1. **初期化**: `GameState.new()` → フィールド作成、キューに5個のランダムTetrominoを入れる。
2. **入力受信**: `handle_event/3` がキーイベントを受け取り、該当関数へ委譲。
3. **落下・固定**: 1秒ごとに `drop/1` を呼び出し。衝突がある場合は `lock_piece/1`（現在ブロックをフィールドに書き込み）→ 次ブロックを current に設定。
4. **行消し**: `clear_lines/1` で消した行数取得、スコア計算・レベル更新。ゲームオーバー判定は `game_over?/1`。
5. **再描画**: LiveView へ状態を assign。 
## 5️⃣ テスト対象関数
- `GameField.new/0`
- `Tetromino.random/0`
- `GameController.valid_position?/3`
- `GameController.drop/1`
- `GameController.rotate/2`

**備考**: 実装はモジュールごとにユニットテストを書き、LiveView では state を assign して描画する。SRS のオフセットは最大9回試行し、最初に合法位置が見つかればそれを採用。
